# Docs Index

This is the public docs map for Webaudit.

Start here when the README tells you the product story and you now want the
next layer of proof, operating detail, and governance boundaries.

> Webaudit is **evidence-first browser automation with recovery and MCP**.
> The pages below help you evaluate that story from first run to side-road
> integrations without pretending the repo is a generic AI bot platform.

If you only need a fast route instead of the full map, choose one lane:

- first run and retained proof
  - [docs/getting-started/human-first-10-min.md](getting-started/human-first-10-min.md)
- coding-agent or MCP fit
  - [docs/how-to/webaudit-for-coding-agents.md](how-to/webaudit-for-coding-agents.md)
- builder/API integration
  - [docs/how-to/api-builder-quickstart.md](how-to/api-builder-quickstart.md)

If you arrived here because you are evaluating a browser-evidence layer for
Codex, Claude Code, OpenHands, OpenCode, OpenClaw, or another AI-agent shell,
finish the first-run lane first, then continue into the second ring below.

![Webaudit proof loop](../assets/storefront/webaudit-readme-hero.svg)
![Webaudit storefront loop](../assets/storefront/webaudit-storefront-loop.gif)

The first-run UI lane stays intentionally strict:
`Quick Launch -> Task Center -> Recovery Center -> Flow Workshop`.

If you arrived here after searching for:

- Codex browser automation
- Claude Code browser automation
- OpenHands browser automation
- OpenCode browser automation
- OpenClaw browser automation
- MCP tools for AI coding agents
- API-first browser automation for AI agents

use this route:

1. [Webaudit for AI Agents](how-to/webaudit-for-ai-agents.md)

Truth boundary:

- Webaudit can act as the browser-evidence substrate for Codex, Claude Code,
  OpenHands, OpenCode, OpenClaw, and other tool-using agent workflows
- Webaudit does **not** claim an official vendor-specific integration or a
  generic coding-agent shell or generic AI assistant shell
